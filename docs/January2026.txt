Clinical Synthetic Data Generation Framework — Contributor Guide (Claude)
This repository benchmarks synthetic tabular data generation methods on clinical datasets using a standardized 5-section notebook pipeline and a modular src/ codebase. This file describes how to extend the framework (new datasets, new models, new preprocessing options) while keeping notebooks backward compatible.

0) Non-negotiable principles
Backward compatibility
Existing notebooks must continue to work with from setup import *.
Prefer adding new functionality under src/ and re-exporting via setup.py.
Reproducibility
All runs must be reproducible via random_state/seeds.
All results must be saved under: results/<dataset_identifier>/<YYYY-MM-DD>/Section-<N>/...
Clear user “knobs”
Users should be able to choose:

(a) which models to run (subset)
(b) smoke vs full tuning trials
© full dataset vs 500-row sample
(d) missing-data strategy: none, median/mode, MICE, and missingness indicator + one-hot encoding (to be implemented)
1) Repository mental model (what exists today)
Notebook pipeline
Section 1: imports setup.py (thin re-export layer) and sets session timestamp
Section 2: dataset ingestion + preprocessing (user edits here)
Section 3: run each model with defaults (demo)
Section 4: Optuna tuning (smoke/full trials)
Section 5: re-run best params + full evaluation
Modular code
setup.py: re-exports functions and patches for notebooks
src/data/: preprocessing utilities
src/models/: model wrappers + factory
src/objective/: objective functions used in tuning
src/evaluation/: quality, TRTS, privacy, batch evaluation
src/visualization/: section-specific plots
2) New “user knobs” to standardize across notebooks
Implement a single configuration block that every notebook can copy/paste into Section 2.1.1. Store it in src/config.py and re-export via setup.py.

Suggested config schema
Python
NOTEBOOK_CONFIG = {
  # Dataset
  "data_file": None,                 # path to CSV
  "dataset_name": None,              # display only
  "dataset_identifier_override": None,

  # Target
  "target_column": None,
  "categorical_columns": [],         # optional override; else auto-detect
  "task_type": "auto",               # auto | classification | regression

  # Subsetting
  "use_row_subset": True,            # True -> sample_n rows
  "sample_n": 500,
  "sample_random_state": 42,

  # Missingness strategy
  "missing_strategy": "none",        # none | drop | median | mode | mice | indicator_onehot
  "mice_max_iter": 10,

  # Encoding
  "encoding_strategy": "auto",       # auto | onehot | ordinal (if needed later)

  # Models selection
  "models_to_run": "all",            # "all" or list like ["ctgan","ctabganplus"]

  # Tuning
  "tuning_mode": "smoke",            # smoke | full
  "n_trials_smoke": 5,
  "n_trials_full": 50,
  "timeout_seconds": None,           # optional
}
Expected behavior
Notebooks must only require users to edit that block and the dataset path/target.
All sections should read from this config to decide:
which models run
whether to subset
which missingness approach is applied
smoke vs full number of trials
3) Preprocessing: required functionality
Create/extend a single entrypoint in src/data/preprocess.py:

preprocess_dataset(df, config) -> (df_processed, metadata)
It must:

Standardize column names (existing utility)
Validate target column exists
Infer categorical columns if not provided
Apply missingness handling based on missing_strategy
Apply encoding if needed (esp. for indicator + one-hot)
Optionally subset to sample_n rows
Return metadata for downstream use:
dataset_identifier
target_column
categorical_columns
task_type
transform_log (what operations were applied)
Missingness strategies (must support)
none: do nothing
drop: drop rows with NA
median/mode: fill numeric/categorical
mice: numeric via IterativeImputer (RF regressor is fine), categorical via mode
indicator_onehot (new):
Add missingness indicator columns for every original column with any NA:
<col>__is_missing as 0/1
Then impute remaining NA:
numeric: median
categorical: mode (or “Unknown”)
Then one-hot encode categorical columns (including target only if target is categorical and the task is not supervised; otherwise keep target untouched)
Important: Never one-hot encode the supervised target used for TRTS/utility evaluation. Keep the target as a single column with stable dtype.

4) Model selection logic (subset of models)
Centralize model lists in src/models/registry.py:

AVAILABLE_MODELS = {"ctgan": ..., "ctabgan": ..., ...}
Implement in src/models/model_factory.py (or wrapper):

resolve_models(models_to_run):
"all" returns all available
list returns intersection and warns on unknown names
Notebook Sections 3–5 should call a single helper:

get_models_to_run(config) (re-export via setup.py)
5) Smoke vs full tuning trials
All tuning cells should compute N_TRIALS via config:

if tuning_mode == "smoke": N_TRIALS = n_trials_smoke
else N_TRIALS = n_trials_full
Also allow:

timeout_seconds passed into study.optimize(..., timeout=...)
6) Consistent target integrity (avoid current failures)
Your logs show:

CTABGAN produced continuous target values (breaks classification metrics)
TVAE produced NaN/inf (breaks correlation)
Add a single sanitizer in src/data/target_integrity.py and call it in:

each model’s .generate(...) wrapper (preferred)
and again in batch evaluation (defensive)
enforce_target_schema(real_df, synth_df, target_column, task_type) -> synth_df_fixed
Rules:

If task_type is classification or if real[target] has small unique count (e.g., <= 20):
Cast synthetic target to numeric if needed
If binary: clamp/round to {0,1}
If multiclass: map to nearest valid class label
If regression: cast to float and allow continuous
Always align dtype with real target dtype
sanitize_numeric(df) -> df
Replace inf/-inf with NaN
Impute NaN in numeric columns with median
Optionally drop rows that still contain NaN in required columns (last resort)
7) Adding new models: PATE-GAN and MEDGAN (immediate)
Where to add
src/models/implementations/
pategan_model.py
medgan_model.py
Register in src/models/model_factory.py and src/models/registry.py
Add hyperparameter search spaces + objective hooks in src/objective/
Ensure evaluate_all_available_models(...) recognizes the new names
Required wrapper interface (must match others)
Each model wrapper must implement:

train(df, **params)
generate(n) -> pd.DataFrame
set_config(params) (optional but recommended)
Store random_state
Respect categorical_columns if relevant
Minimal tuning support
Provide at least:

epochs
batch_size
learning rates (if applicable)
architecture dims (if applicable)
Notes
PATE-GAN has privacy budget / accountant parameters; ensure they are logged and saved.
MEDGAN often expects discrete/encoded inputs; integrate with preprocessing/encoding pipeline cleanly.
8) Notebook contract (what Section 2 must output)
After Section 2 runs, these globals must exist for Sections 3–5:

data (processed DataFrame)
original_data (copy before modeling, but after preprocessing is fine if consistent)
target_column and TARGET_COLUMN (same value)
DATASET_IDENTIFIER
categorical_columns
NOTEBOOK_CONFIG
Prefer producing them via a single helper in setup.py:

load_and_preprocess_from_config(NOTEBOOK_CONFIG)
9) Results and logging requirements
Every section/model must write:

parameters used (json/csv)
generated synthetic sample (optional; at least summary stats)
evaluation summaries
Use get_results_path(dataset_identifier, section_number, model_name=None) everywhere.

10) Definition of done (for any PR/change)
Existing notebooks run without modification
New config block works on at least one dataset notebook end-to-end
Subset model selection works (e.g., only CTGAN + PATE-GAN)
Smoke vs full trials works
Missingness strategies all run (including indicator_onehot)
Target integrity prevents CTABGAN continuous-target failures
NaN/inf sanitization prevents correlation failures
Results saved under the standard folder structure
